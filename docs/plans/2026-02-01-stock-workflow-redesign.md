# Stock Workflow Redesign

**Date**: 2026-02-01
**Status**: Approved
**Goal**: Simplify stock adjustment flows for tracked items (batch/serial) following "mom & pop" usability principles.

---

## Design Philosophy

### The "Mom & Pop" Principle
Stock adjustments should feel like counting inventory on a clipboard - simple, familiar, and forgiving.

### Three User Mindsets

| Mindset | What they want | Example |
|---------|---------------|---------|
| **Quick** | "Just take 5 off" | One-tap adjustment, system handles the rest |
| **Specific** | "Take from batch X" | Override auto-pick, choose exact source |
| **Audit** | "What happened?" | View history of all movements |

### Auto-Pick Strategy (FIFO/FEFO)
- System always suggests which batch/serial to use
- **Batches**: Earliest expiry date first, then oldest received
- **Serials**: Oldest added first (by created_at)
- User can always override with one extra tap

---

## Actions

### Two Primary Actions

| Action | Intent | Stock Effect | Tracking |
|--------|--------|--------------|----------|
| **Stock Out / Stock In** | Permanent change | Immediate +/- | Reason logged |
| **Borrow Out** | Temporary loan | Reserved until returned | Assignee + due date |

### Terminology
- **Stock Out** = Remove stock (consumption, damaged, expired, correction)
- **Stock In** = Add stock (received, returned, correction)
- **Borrow Out** = Temporary checkout to person/job/location

## Auto-Pick (FIFO/FEFO) Logic

### For Batches
```sql
SELECT * FROM lots
WHERE item_id = ? AND quantity > 0 AND status = 'active'
ORDER BY
  expiry_date ASC NULLS LAST,  -- Earliest expiry first
  created_at ASC               -- Then oldest received
```

### For Serials
```sql
SELECT * FROM serial_numbers
WHERE item_id = ? AND status = 'available'
ORDER BY created_at ASC        -- Oldest first
```

### Multi-Batch Deduction
When quantity exceeds a single batch:
1. Start with FIFO-suggested batch
2. Deduct available amount
3. Move to next batch automatically
4. Continue until quantity fulfilled
